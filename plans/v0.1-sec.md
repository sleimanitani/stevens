# Build Plan — v0.1-sec: Security Agent foundation

**Goal:** Land the Security Agent as sole secret broker. Every agent that touches OAuth tokens, payment instruments, or API keys will be built on top of it. No such agent ships before this milestone is complete.

**Status:** in-progress — started 2026-04-22
**Charter refs:** `STEVENS.md` §3 (security architecture), §3.12 (intended sequence), §3.11 (locked decisions)
**Blocks:** Email PM (PRD Day 4), all future credential-using agents.

## Protocol — Security Agent RPC (v1 draft)

*Stabilize and move to `docs/protocols/security-agent.md` once step 7 ships.*

- **Transport:** Unix domain socket at `/run/stevens/security.sock`. No TCP. Permissions `0660` owned by user `stevens`; each caller container mounts the socket with the matching group.
- **Framing:** length-prefixed msgpack. `uint32_be` payload length, then msgpack bytes. Max payload 1 MiB in v1.
- **Request envelope:**
  ```
  {
    "v": 1,
    "caller": "<agent_name>",       # e.g. "email_pm"
    "nonce": "<32 random bytes, base64>",
    "ts":    <unix_seconds>,
    "capability": "<namespace.name>",  # e.g. "gmail.send_draft"
    "params": { ... },              # capability-specific
    "sig":   "<ed25519 sig, base64>"
  }
  ```
- **Signature scope:** Ed25519 over `canonical_msgpack({v, caller, nonce, ts, capability, params})` with canonical = deterministic key order (lexicographic), no extensions, fixed int widths. Full rules in `docs/protocols/security-agent.md` once step 3 ships.
- **Replay protection:** server keeps an in-memory nonce LRU (capacity ≥100k, TTL 5 min). Clock skew tolerance ±60 s; requests outside the window are rejected.
- **Response — ok:**
  ```
  { "ok": true, "result": { ... }, "trace_id": "<uuid>" }
  ```
- **Response — error:**
  ```
  { "ok": false, "error_code": "<AUTH|DENY|NOTFOUND|RATE|INTERNAL>", "message": "<human>", "trace_id": "<uuid>" }
  ```
- **Audit:** every request produces one JSONL line in `/var/lib/stevens/audit/YYYY-MM-DD.jsonl`: `{ts, trace_id, caller, capability, account_id, param_hashes, outcome, error_code?, latency_ms}`. Sensitive param values are SHA-256 hashed; non-sensitive (account_id, capability) logged in clear. List of sensitive params per capability is declared in the capability's registration, default-sensitive for anything not listed.

## Risks

- **Canonical msgpack drift between languages.** Node clients (WhatsApp adapter, eventually) must produce the same bytes Python does. Mitigation: strict spec in `docs/protocols/security-agent.md`, cross-language golden fixture, don't use msgpack features (ext types, str8, …) that differ between libs.
- **Nonce LRU exhaustion under pump.** v0.1 scale is tiny; if it ever becomes real, migrate to a time-bucketed Bloom filter or Redis-backed set. Documented risk, not solved now.
- **Policy yaml fat-fingered → lockout.** Refuse to start on malformed yaml (loud) rather than default-allow (silent). Always.
- **UDS permission mishap.** The socket is the trust boundary — wrong group perms = either nothing works or everything can connect. Integration test in step 7 verifies permissions.

- [ ] Email PM can draft a Gmail reply without ever holding a raw OAuth token in its process memory.
