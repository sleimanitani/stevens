# Build Plan — v0.3.1-web: Arachne agent + network.fetch / network.search

**Goal:** ship Stevens's web reach. Synchronous capabilities for ReAct agents (`network.fetch`, `network.search`) and an async-path agent — **Arachne** — for event-driven workloads. The PDF reader's regression-corpus exercise is the integration test.

**Status:** in-progress — started 2026-04-30
**Charter refs:** STEVENS.md §1.1 (Names), §3 (Security architecture). Cross-cutting docs: `agent-isolation.md`, `approvals.md`, `privileged-execution.md`.
**Predecessors:** v0.3-installer-and-approvals (`c2f0929`).
**Blocks:** future research / subject agents that need outbound web; the PDF reader's full 4/4 acceptance gate (waits on tesseract + corpus).

## Architectural posture

Two paths to the same outbound, both broker-mediated:

- **Synchronous path** — used by ReAct/LangChain agents (Email PM and friends today). Skill makes a sync tool call → Enkidu's `network.fetch` or `network.search` capability → bytes back. No bus round-trip. This is the load-bearing path for v0.3.1's PDF corpus exercise.
- **Async path (preferred shape long-term)** — agents publish `web.fetch.requested.*` / `web.search.requested.*`; Arachne subscribes, calls the same Enkidu capabilities under a worker pool, publishes response events. Future event-driven workloads (research crawlers, scheduled fetches) use this. ReAct agents can't yield mid-thought to wait on a bus event, which is why the sync path exists; a future event-driven agent shape will use Arachne natively.

Both paths converge at the same Enkidu capabilities, so cache / rate-limit / quota are shared by construction. Arachne does **not** sit between the skill and Enkidu — it's a parallel handler for events, not a synchronous proxy.

## Scope

In this milestone:

1. `shared/outbound/web.py` — `WebClient` (httpx-backed) + URL validator (scheme allow-list, private-CIDR deny-list, max size, timeout, redirect policy).
2. Modular search backend layer with **Brave** as the default. `SearchBackend` Protocol so Tavily / SearXNG / DuckDuckGo drop in later.
3. In-memory TTL cache (per-fetch and per-search-query) + per-domain token-bucket rate limiter, both inside Enkidu.
4. `network.fetch` and `network.search` capabilities in Enkidu.
5. Skill wrappers `skills/src/skills/tools/web/{fetch,search}.py` — synchronous, `scope: shared, safety_class: read-only`.
6. Event schemas in `shared.events` for fetch/search request/response.
7. **Arachne** — `agents/src/agents/web/`, display name registered in `STEVENS.md §1.1`, code id `web`. Subscribes to `web.{fetch,search}.requested.*`. 4 concurrent workers. Calls the same Enkidu capabilities; publishes response events.
8. `stevens secrets add web.brave.api_key --from-stdin` documented onboarding.
9. `scripts/test_pdf_reader_on_corpus.py` — downloads 4-5 public PDFs via the synchronous skill, runs `read_pdf` on each, prints a per-file report.
10. Doc updates: `STEVENS.md §1.1` adds Arachne; `docs/architecture/agent-isolation.md` adds notes (a) future shared cache via Postgres + Enkidu-mediated ACL, (b) Browser Harness as a future reference for browser-automation needs (separate from web HTTP).

Out of scope (deferred):

- Postgres-backed cache (in-memory only this milestone). Doc note in agent-isolation.md captures the future shape (Enkidu-mediated `web.cache.{get,put}` capabilities, ACL enforced at the broker).
- Authenticated fetches (cookies, login flows).
- POST / PUT / DELETE — GET only in v1.
- Browser automation (Playwright/Selenium-shaped). Distinct problem; future milestone.
- Real Brave API key; tests use httpx.MockTransport.

## Risks

- **Brave API quota.** 2k req/mo free tier is shared across all of Stevens's search traffic. Per-domain rate limit at the Enkidu level guards against runaway loops; budget alarm is a v0.4 concern.
- **In-memory cache lost on Enkidu restart.** Acceptable for v0.3.1 — the alternative (Postgres-backed cache with ACL) is a real workstream. Documented as a known limitation.
- **DNS resolution timing for the private-CIDR check.** The validator checks the resolved IP against private-CIDR ranges before issuing the request, but DNS records can change between check and request (TOCTOU). Mitigation: do the resolution once and pass the resolved IP into httpx via a custom transport (or fall back to URL hostname rejection for known internal patterns). Documented in the validator module.
- **Skill latency on cache miss.** Synchronous skill blocks until the fetch completes. 30s timeout means worst-case the agent's tool call takes 30s. Acceptable for ReAct agents that don't have hard latency budgets; the async path via Arachne is the right shape if latency matters.
- **Public PDFs change.** The corpus URLs may go stale or change content. Mitigation: cache PDFs locally on first run (sha256-pinned) so the regression test is reproducible. Scoped to the script, not in the cache primitive.

- [ ] `network.fetch` rejects all RFC-1918 + link-local + loopback + IPv6-ULA destinations.
- [ ] `network.search` returns normalized `{results: [{title, url, snippet}]}` against a Brave-shaped MockTransport response.

- [ ] Per-domain rate limiter denies past budget; recovers after refill.
- [ ] Skill `web_fetch` returns the body on the synchronous path; `web_search` returns normalized results.
- [ ] Arachne handles `web.fetch.requested.*` events and publishes corresponding `web.fetch.response.*` events with the original `request_id`.
- [ ] Concurrent worker pool caps in-flight fetches at 4 (semaphore-asserted in tests).
