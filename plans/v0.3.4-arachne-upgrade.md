# Build Plan — v0.3.4-arachne-upgrade

**Goal:** validate the search-backend Protocol with three more implementations (Tavily, Exa, Firecrawl) and add LLM-based content compression for fetched pages — the two patterns most worth borrowing from Hermes for Arachne.

**Status:** in-progress — started 2026-05-01
**Predecessors:** v0.3.1-web (`8d1f64a`), v0.3.3 (`751d86d`).

---

## Scope

1. **Three new search backends** — Tavily, Exa, Firecrawl (search). Each is one new module under `shared/outbound/search/<backend>.py`, registers via the existing `register_backend` hook, normalizes responses to the same `SearchResults` shape. Sealed-store key naming: `web.<backend>.api_key`.
2. **`network.compress` capability** — given an HTML body + a query / hint, returns a compressed extract. Implementation strategy: pluggable LLM backend (defaults to Anthropic Claude Haiku with API key in sealed store as `compress.anthropic.api_key`). For v0.3.4 the only built-in compressor is "Anthropic"; future Gemini / local Qwen alternatives drop in behind the same Protocol.
3. **`web_compress` skill** — synchronous wrapper around `network.compress` for ReAct agents.
4. **Fetch tool integration** — `web_fetch` skill grows an optional `compress_with_query: str` arg. When set, after fetching it pipes the body through `network.compress` and returns the compressed result alongside the raw.
5. **Lazy-import optimization for heavy backends** — Firecrawl SDK is the canonical case; for v0.3.4 we use plain httpx for all three (no SDK dependency), so this becomes a doc note rather than a concrete change. Documented as a follow-up if SDK use becomes needed.

Out of scope: real Anthropic / Tavily / Exa / Firecrawl API runs (mocked-transport tests only). Real-network exercise via the PDF corpus script remains the manual integration.

## Steps

> `[ ]` todo · `[~]` in progress · `[x]` done.

### Step 1 — Tavily, Exa, Firecrawl backends [x]

**Files:**
- `shared/src/shared/outbound/search/tavily.py` — Tavily Search API. POST to `https://api.tavily.com/search` with `{api_key, query, max_results}`; normalize response.
- `shared/src/shared/outbound/search/exa.py` — Exa neural-search API. POST to `https://api.exa.ai/search` with `x-api-key` header; normalize.
- `shared/src/shared/outbound/search/firecrawl.py` — Firecrawl Search API. POST to `https://api.firecrawl.dev/v1/search` with bearer token; normalize.
- `shared/src/shared/outbound/search/__init__.py` — import the three new modules so they self-register.
- `shared/tests/test_search_backends.py` — 9 tests (3 per backend): happy path, 401, malformed response. All `httpx.MockTransport`.

### Step 2 — Compressor backend layer [x]

**Files:**
- `shared/src/shared/outbound/compress/__init__.py` — `CompressBackend` Protocol, `CompressResult` dataclass (compressed_text, original_chars, compressed_chars, ratio), registry pattern matching the search-backend layer.
- `shared/src/shared/outbound/compress/anthropic.py` — Anthropic-Haiku-based compressor. Calls `https://api.anthropic.com/v1/messages` with the Haiku model + a fixed compression prompt parameterized by the query. Strips HTML tags first (`html.parser`-based) so the model gets cleaner input.
- `shared/tests/test_compress_anthropic.py` — 4 tests: happy path, 401, missing api key error, ratio calculation correct.

### Step 3 — `network.compress` capability + skill [x]

**Files:**
- `security/src/demiurge/capabilities/network.py` — add `network.compress` capability. Reads `compress.<backend>.api_key` from sealed store; selects backend via env `DEMIURGE_COMPRESS_BACKEND` (default `anthropic`); audits `query` hashed, `backend` clear.
- `skills/src/skills/tools/web/compress.py` — `web_compress` skill, synchronous, wraps `network.compress`.
- `skills/registry.yaml` — register as `scope: shared, safety_class: read-only`.
- `security/tests/test_capability_compress.py` — 4 tests: happy path, missing api key, cache (no — compression isn't cached; each call has a different query intent), unknown backend.

### Step 4 — Fetch tool integration [x]

**Files:**

- `skills/tests/test_web_skills.py` — 1 new test: fetch with compress_with_query → returns both fields.

## Out-of-scope follow-ups (noted)

- **SDK lazy-import pattern** — when we adopt a backend that ships a heavy SDK (e.g. official Firecrawl SDK rather than raw HTTP), do the lazy-proxy thing Hermes does in `tools/web_tools.py`.
- **Compressor on Arachne (async path)** — Arachne could optionally compress before publishing the fetch response. Ship after the first event-driven consumer that asks for it.
- **Compressor caching** — at first glance compression is per-(url, query) and worth caching. Defer until we measure cache-hit rate in practice.
