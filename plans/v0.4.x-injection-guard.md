# Build Plan — v0.4.x-injection-guard

**Goal:** ship a shared prompt-injection scanner so any user-supplied or third-party markdown that gets injected into agent prompts (CLAUDE.md, SOUL.md, USER.md, project AGENTS.md, fetched web content if used in-prompt, etc.) is screened first. Borrowed from Hermes's `agent/prompt_builder.py` pattern. Also: document the SOUL.md / USER.md split for the future interface agent.

**Status:** in-progress — started 2026-05-01
**Predecessors:** v0.4.1-channels-framework (`31f464f`).
**Reference:** Hermes's `agent/prompt_builder.py` injection-detection (per the read-through report).

---

## Scope

1. **`shared/prompt_safety.py`** — `scan_for_injection(text) -> ScanResult` (severity, markers found, redacted_text). Detects:
   - "Ignore previous instructions" + variants
   - System-prompt impersonation (`</system>`, `<|system|>`, `[SYSTEM]`)
   - Tool-call injection markers (`<tool_call>`, `<function_call>`)
   - Hidden HTML divs / CSS-display-none content (a common exfil vector)
   - Credential-pattern reads ("show me your API key", "what's in your env", etc.)
   - Suspiciously long base64 / hex blobs (potential payloads)
2. **`safe_load_user_markdown(path) -> str | RaisesScanError`** — file loader that scans before returning. Strips YAML frontmatter (callers don't want it in the prompt anyway). Used by future SOUL.md / USER.md / AGENTS.md loaders.
3. **Doc note in `docs/architecture/agent-isolation.md`** — short subsection on the SOUL.md vs USER.md vs AGENTS.md split for the future interface agent (per the OpenClaw + Hermes patterns; we already noted SOUL.md in `CLAUDE_openclaw_evaluation.md` as v0.2+ work).

Out of scope:
- Wiring the scanner into any existing agent (no agent currently injects user-supplied markdown into its prompt; this lands when the interface agent ships).
- LLM-based injection detection — v0.4.x is regex + structural patterns only. Hermes uses both.
